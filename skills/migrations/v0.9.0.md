# v0.9.0 Migration: Technical Knowledge Map + Agent Rule Refresh

This migration introduces technical knowledge maps for codebase work and makes
agent rule refresh part of the upgrade path.

## What's New

### Searchable Technical Knowledge Maps

GBrain can now store:

- `systems/` pages for repos, runtimes, and subsystems
- `codemap` frontmatter on concept/system pages for exact file and symbol pointers
- shared vocabulary mappings between user language and implementation language

The key difference from earlier docs-only guidance: codemap entries are now
indexed for search, so symbol and path lookups can find the right page before
the agent starts broad repo exploration.

### Upgrade Refreshes Agent Rules

After a successful `gbrain upgrade`, the CLI now reruns
`gbrain setup-agent --skip-mcp` when Claude Code or Codex is installed.
Existing users no longer need to remember a separate step to pick up updated
brain rules, and MCP registration is left alone.

## Agent Instructions

When presenting this update, say something like:

"v0.9.0 gives your brain a technical map. I can store exact files, symbols,
entry points, and vocabulary for a codebase, then search that map before
opening source. That makes repeat architecture work much faster."

If the user wants to apply it:

1. Create or update a `systems/<name>.md` page for the repo or subsystem
2. Add `codemap` pointers to the concept or system pages that matter
3. Verify each pointer against the current code before saving it
4. Run `gbrain embed --stale` if embeddings are enabled so new technical chunks get vector coverage
5. Reuse those pages as the first lookup step on future technical questions
